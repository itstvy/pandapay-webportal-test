# 🚀 7-Day Robot Framework Practical Learning Roadmap for Automation Testers

This roadmap is designed for testers (especially manual testers transitioning to automation) who want to learn Robot Framework step by step and apply it in real-world projects (UI, API, data-driven testing, CI/CD integration, etc.)

---

## 🟢 Day 1: Foundation & Core Concepts

### 🎯 Goal:
- Understand the architecture of Robot Framework
- Write your first test case

### 🧠 Recommended Prompts:
1. `Teach me Robot Framework from A to Z, including installation, structure, basic keywords, and how to run the first test.`
2. `Explain the structure of a .robot file and the roles of *** Settings ***, *** Test Cases ***, and *** Keywords ***.`
3. `Compare Robot Framework with Selenium WebDriver using Python. What are the pros and cons for UI testing?`

---

## 🟡 Day 2: Basic UI Testing with SeleniumLibrary

### 🎯 Goal:
- Automate basic UI interactions: login form, navigation, input handling

### 🧠 Recommended Prompts:
4. `Write Robot Framework test cases to verify login functionality with valid and invalid credentials.`
5. `Show me how to use SeleniumLibrary to interact with buttons, inputs, and checkboxes.`
6. `How do I wait for an element to appear on screen in Robot Framework?`

---

## 🟠 Day 3: Advanced UI Flow + Element Handling

### 🎯 Goal:
- Build multi-step UI flows and handle advanced web components

### 🧠 Recommended Prompts:
7. `Write a test case for the flow: Login → Navigate to Device Page → Validate device list.`
8. `Explain how to handle dropdowns, alerts, iframes, and new browser tabs using Robot Framework.`

---

## 🔵 Day 4: REST API Automation using RequestsLibrary

### 🎯 Goal:
- Test REST APIs including authentication, status validation, and token reuse

### 🧠 Recommended Prompts:
9. `Introduce RequestsLibrary and show how to send GET, POST, and PUT requests in Robot Framework.`
10. `Write a test case for API login, retrieve access token, and reuse it in subsequent API calls.`
11. `How can I validate response body, status code, and headers in Robot Framework API testing?`

---

## 🔴 Day 5: Data-Driven Testing + Custom Keywords

### 🎯 Goal:
- Loop through test data, define reusable keywords, extend Robot with Python

### 🧠 Recommended Prompts:
12. `Does Robot Framework support data-driven testing? Give me an example.`
13. `How do I create custom Python keywords and use them in .robot files?`
14. `Show me how to write an API test case with Bearer Token authentication.`

---

## 🟣 Day 6: Test Organization, Tagging & Modularity

### 🎯 Goal:
- Structure test suites, reuse resources, and manage tests efficiently

### 🧠 Recommended Prompts:
15. `How can I organize tests across multiple .robot files and reuse keywords?`
16. `I want to run tests by tag, group, or test name. How can I do that in Robot Framework?`
17. `How do I manage Robot Framework test cases in a real project using Git?`

---

## 🟤 Day 7: Reporting, CI/CD Integration, and Final Practice

### 🎯 Goal:
- Generate better reports and connect Robot Framework with CI/CD pipelines

### 🧠 Recommended Prompts:
18. `How do I generate beautiful reports for Robot Framework (e.g., Allure Report)?`
19. `Can I run Robot Framework tests on cloud browsers like BrowserStack or LambdaTest? How to configure it?`
20. `Write a real-world Robot Framework test suite for: Login → View product list → Create order → Confirm.`

---
